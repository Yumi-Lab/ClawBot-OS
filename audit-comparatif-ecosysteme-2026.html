OpenClaw Gateway (Node.js, port 18789) │ ├─ Channel Adapters → Telegram, Discord, WhatsApp, Slack, Signal, iMessage, QQ, LINE, WeCom ├─ Agent Loop → Context builder → LLM (any provider) → Tool registry → Memory update ├─ Browser Tool → Managed Chromium ou extension relay ├─ Memory → Session + Daily logs + Long-term Markdown + Vector search ├─ Skills → 10,700+ sur ClawHub, SKILL.md avec YAML frontmatter ├─ Heartbeat → Daemon autonome toutes les 30 min (HEARTBEAT.md) ├─ ACP → Agent Client Protocol (JSON-RPC 2.0 over NDJSON/stdio) └─ Plugins → Hot-loading via package.json `openclaw.extensions` │ Ressources: 2-4 GB RAM, Node.js 22+, SSD recommandé Plateformes: Linux, macOS, WSL2, Raspberry Pi, Docker, Android Sécurité: 7+ CVE, 42K instances exposées, prompt injection persistant

6. Analyse de sécurité

Audit des vecteurs d'attaque, protections et vulnérabilités

D OpenClaw (ex-Moltbot) — CVE multiples

7+ CVE publiés. CVE-2026-25253 (CVSS 8.8) : RCE, compromission gateway. ClawJacked : sites malveillants hijackent l'agent local. 42K instances exposées. Skills malveillants (820+ sur ClawHub). Microsoft, Kaspersky, CrowdStrike ont publié des advisories.

D OpenClaw — Prompt injection

La mémoire persistante aggrave le risque : instructions malveillantes dans PDFs/emails restent dormantes et se déclenchent des jours plus tard (« stateful delayed-execution attacks »). Cross-user privacy breaches documentés.

C PicoClaw — Auth

API keys en clair (chmod 600). Pas de chiffrement. Pas d'auth sur gateway. Risque de prompt injection via messages entrants. Pas de blocage de commandes dangereuses.

D ClawbotCore — Isolation

Tourne en root sans sandbox. L'outil bash exécute n'importe quelle commande (sauf services protégés). Prompt injection → RCE potentiel. Pas d'auth sur API locale.

B Cloud (openjarvis.io)

bcrypt + JWT HS256 (72h). Sub keys uniques. Rate limiting quotidien. Anti-récursion tunnel. Admin protégé. Mais : MAC comme seul ID device (spoofable).

A Claude Code — Sandbox

Sandbox OS natif (Seatbelt/bubblewrap). Isolation filesystem + réseau. 4 modes de permissions. Hooks pre/post. Réduit 95% surface d'attaque. Gold standard.

D Kimi Claw — Souveraineté & CVE

Cloud chinois (Moonshot AI, Pékin). Données soumises à la PIPL (loi chinoise) et à la Loi sur la Sécurité Nationale. Pas de conformité RGPD. 7+ CVE hérités du fork OpenClaw (mêmes vecteurs : ClawJacked, ClawHavoc, prompt injection). Alerte IAPS 2026 : risque de collecte de données sensibles via skills tiers.
