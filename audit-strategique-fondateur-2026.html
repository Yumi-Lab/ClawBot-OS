1 · Thèse de victoire

Pourquoi ClawBot peut gagner

Pas une liste de features — un argument. Trois moats structurels, les conditions nécessaires pour qu'ils tiennent, et les scénarios qui les neutralisent.

Thèse : ClawBot gagne dans le segment "IA personnelle souveraine" parce que les trois alternatives (cloud chinois, cloud US, DIY technique) ont chacune un défaut structurel insurmontable pour les familles et PME européennes soucieuses de leur vie privée. Ces défauts sont permanents — pas des retards de roadmap. Le hardware physique + l'exécution locale créent un moat qu'aucun logiciel seul ne peut répliquer à court terme.
01
Hardware physique — L'objet dans ta maison
Kimi Claw, OpenClaw, Claude Code sont tous cloud ou logiciel. Un boîtier physique à 29€ crée un lien émotionnel, une présence dans la maison, et un coût de switch asymétrique. L'utilisateur ne "désabonne" pas son assistant — il retire un objet de son salon.
Durabilité : Forte— nécessite une supply chain, irréplicable en logiciel
02
Exécution locale — L'IA qui fait vraiment des choses
bash, python, ssh, fichiers — exécutés sur votre propre machine, pas sur un sandbox cloud. L'IA contrôle réellement votre Pi, vos dossiers, vos scripts. Kimi Claw ne peut pas lancer un script Python qui tourne 24h sur votre NAS. ClawBot oui.
Durabilité : Forte— intrinsèque au modèle physique local
03
Souveraineté — Vos données ne vont pas en Chine
Le rapport IAPS de mars 2026 sur Kimi Claw est cité par 3 médias tech européens. Les lois chinoises de surveillance nationale s'appliquent à Moonshot AI. Anthropic US + exécution locale = le seul stack IA personnel conforme RGPD au niveau enterprise. Argument béton côté PME/legal/médical.
Durabilité : Moyenne— Kimi peut ouvrir des serveurs EU (12-18 mois)
Conditions nécessaires pour que la thèse tienne
✓ Doit être vrai La sécurité RGPD reste un critère d'achat réel pour les familles FR/EU (pas juste les DSI)
✓ Doit être vrai Kimi n'ouvre pas de serveurs EU avant 12 mois — fenêtre pour s'installer
✓ Doit être vrai L'exécution locale reste une valeur ajoutée perçue (pas juste une technicité)
⚠ Risque Apple/Google n'intègrent pas d'IA locale performante dans leurs OS d'ici 18 mois
⚠ Risque Anthropic ne multiplie pas ses prix par 3+ (rendrait l'économie insoutenable sans K2.5)

6 · Scalabilité & Dette technique

Les plafonds actuels et ce qui se casse en premier

Pas "SQLite → PostgreSQL planifié". Des chiffres. Combien d'utilisateurs simultanés avant la panne ? Où est le bottleneck ? Combien de jours pour corriger la dette critique ?

Thèse : La stack actuelle tient la beta (<100 utilisateurs actifs simultanés) mais ne passe pas 500 utilisateurs sans deux migrations critiques : SQLite → PostgreSQL et proxy LLM asynchrone. Ces migrations doivent démarrer dès Epic 4, pas après la première panne en production.
Single Points of Failure (SPOF)
SPOF #1 — Critique
SQLite mono-fichier
Un seul fichier /var/lib/clawbot-cloud/db.sqlite3 sur un seul VPS. Corruption → perte totale. Écriture concurrente >10 req/s → WAL lock timeout. Pas de backup automatisé confirmé.
Fix : Migration PostgreSQL + backup daily S3. Effort : 5j
SPOF #2 — Critique
Proxy LLM synchrone
Le proxy LLM dans clawbot-cloud est synchrone (WSGI/threads). Une requête Opus bloquée >30s bloque un thread. À 20+ requêtes simultanées longues → pool épuisé → timeout pour tous les autres utilisateurs. Pas de queue, pas de circuit breaker.
Fix : Migration async (FastAPI + asyncio natif). Effort : 5j
SPOF #3 — Majeur
VPS unique sans redondance
87.106.137.67 = un seul serveur IONOS. Maintenance planifiée ou crash → service total indisponible. Pas de CDN, pas de load balancer, pas de health check externe.
Fix : Health check UptimeRobot (gratuit) + VPS standby. Effort : 1j

9 · Scénarios

Ce qui peut tout changer — dans les 18 prochains mois

Quatre scénarios adversariaux. Pour chacun : probabilité, impact, réponse disponible. Un audit sans scénarios est un audit de beau temps.

💸
Kimi baisse à 2€/mois
Probabilité : Moyenne
Moonshot AI peut sacrifier sa marge pendant 12-18 mois pour capturer du marché (ils ont $1Md de financement). Si Kimi Claw passe à 2€/mois, notre plan Starter à 7.99€ devient difficile à justifier pour un utilisateur non-sensible RGPD.
Réponse : Accentuer l'axe hardware physique (impossible à répliquer en SaaS), l'exécution locale réelle (pas du sandbox cloud), et le RGPD. Baisser notre plan Free à 0€ avec publicité locale comme alternative. Intégrer K2.5 pour réduire nos coûts et potentiellement baisser les prix.
📈
Anthropic multiplie ses prix par 3
Probabilité : Faible
Si Anthropic triple ses prix API, notre COGS token explose. Le plan Starter à 7.99€ devient instantanément déficitaire même avec throttle agressif. Sonnet output à $45/M tokens = fin du modèle économique actuel sans migration vers un autre provider.
Réponse : Intégration multi-provider (K2.5, DeepSeek R2, Llama 4 local) est l'assurance-vie contre cette dépendance. Epic 7 devient alors non plus une optimisation mais une survie. Hardware Ollama pour modèles locaux comme fallback ultime.
🍎
Apple Intelligence devient vraiment capable
Probabilité : Élevée (18 mois)
En 2027, Apple Intelligence intégré à iOS/macOS avec Private Cloud Compute pourrait offrir "IA locale + cloud privé Apple" à 0€ de coût marginal pour les 60% de foyers FR avec iPhone. L'argument "local + privé" serait absorbé par l'écosystème Apple.
Réponse : ClawBot = IA qui exécute du vrai code sur votre infrastructure physique. Apple ne donnera jamais accès à votre NAS, vos scripts bash, votre home server. Pivoter vers le segment "Maker/Pro" si le grand public va vers Apple. La fenêtre est de 18 mois — s'installer maintenant.
🤝
Offre d'acquisition ou partenariat stratégique
Probabilité : Faible (6 mois)
Un acteur comme OVHcloud, Scaleway, ou un distributeur hardware FR/EU pourrait vouloir acquérir ou distribuer ClawBot pour compléter son offre "cloud souverain français". Scénario positif mais qui nécessite d'avoir une traction prouvée (500+ abonnés, MRR croissant).
Réponse : Préparer la narrative "actif stratégique souveraineté numérique française". BUSL → Apache 2036 si acquisition. Documenter ARR, churn, NPS dès la beta pour crédibiliser. Ne pas chercher activement avant €10K MRR.
