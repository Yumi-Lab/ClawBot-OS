Principe fondateur

Le bridge est un tuyau. L'intelligence vient du mode (Core ou Core-Agent). Le dashboard est une fenetre read-only. Une seule interface, identique dans tous les modes. On observe, on n'intervient pas.

1. Deux modes, une seule interface

👤
Mode 1

Bot Personnel

Bridge de son propre WhatsApp. Self-chat pour piloter le boitier.

Tu es automatiquement admin.

Acces complet : slash commands, mode switch, vault (avec 2FA optionnel), outils, config.

admins = ton numero (auto)
allow_from = ton numero
🤖
Mode 2

Bot Autonome

Numero dedie. L'AI repond a tout le monde (ou whitelist). Certains numeros sont admin pour piloter.

Tu definis les numeros admin.

Les admins pilotent. Les autres recoivent les reponses de l'AI selon sa programmation.

admins = +336..., +331...
allow_from = * (ou whitelist)
blacklist = +339... (optionnel)

Regle cle : Bot Personnel = Admin automatique

En mode Bot Personnel, le numero de l'utilisateur est automatiquement admin. Pas de configuration necessaire. En mode Bot Autonome, il faut definir explicitement au moins un numero admin.

3. Securite

🔒 Vault 2FA (optionnel)

Quand active, l'acces au vault via WhatsApp necessite un mot de passe. L'AI demande : "Envoyez votre mot de passe vault pour continuer." Le telephone est securise, mais double securite en cas de vol/compromission.

vault_2fa = on | off
Desactivable par l'utilisateur au choix

🚫 Tool restriction par role

Les non-admins ne peuvent PAS acceder aux outils systeme (bash, ssh, write_file, vault). Configurable : quels outils sont disponibles pour la whitelist. Empeche "donne-moi ta localisation" ou "montre-moi tes mots de passe".

user_tools = web_search (par defaut)
admin_tools = * (tous les outils)

📝 System prompt par role

Les non-admins recoivent un system prompt cadre : "Tu es un assistant. Tu ne partages jamais de donnees sensibles, mots de passe, tokens ou informations systeme." Configurable par l'admin via le dashboard.

🛡 Blacklist

Numeros explicitement bloques. Le bridge ignore completement leurs messages. Utile en mode allow_from = * pour bloquer du spam.

blacklist = +339..., +447...

5. Architecture — flux de decision

Message WhatsApp entrant | v [Blacklist?] ── oui ──> IGNORE | non v [Admin?] ── oui ──> ACCES COMPLET | Slash commands, tous outils, vault (+2FA opt.) non System prompt normal v [Whitelist?] ── non et allow_from != * ──> IGNORE | oui (ou allow_from=*) v ACCES LIMITE Conversations normales Outils restreints (user_tools) System prompt cadre (pas de secrets) Pas de slash commands Dashboard observe tout en read-only (identique dans tous les cas)
